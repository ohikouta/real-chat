# Project Assessment

この文書は、2026-03-24 時点の `Real Chat` を「今どこまでできていて、次に何を優先すべきか」の観点で整理するための現状評価です。

## 現在の到達点

- Vue 3 + Firebase で最低限のチャットアプリとして動作する
- 認証、ユーザー一覧、プロフィール、1対1 DM、スレッド投稿、スレッド詳細、コメント投稿まで一通り存在する
- Firestore Security Rules と emulator ベースの検証導線が入っている
- `npm run lint` `npm run build` `npm run test:rules` の基本的な検証導線が揃っている
- ドキュメントは Firestore / routes / emulator acceptance まで揃っている

## 強み

### 1. 学習用プロジェクトとしての題材がよい

- 認証、Firestore、Storage、Routing、Security Rules まで一通り触れている
- 画面実装だけでなく、データ設計や権限制御まで学べる構成になっている
- `threads` と `directMessages` が並立しており、用途の異なるデータモデリングを比較しやすい

### 2. 最低限の運用フローが整っている

- GitHub Issue / PR を基準に進める運用に寄せられた
- Copilot レビュー反映、Obsidian / session-memory 同期など、作業の痕跡を残す前提ができた
- worktree ベースで Issue 単位に分ける流れが定着しつつある

### 3. 設計と実装の往復ができる状態にある

- `docs/db/firestore.md` と `docs/routes.md` が、実装変更時の基準として機能し始めている
- 実装しながら設計を更新する流れを作れた

## 現時点の弱み

### 1. ドキュメントの層がまだ分散している

- README だけでは「何ができて、何が未完か」が十分に伝わらない
- 設計文書、運用メモ、振り返りが別れており、初見で全体像を掴みにくい

### 2. 公開運用の前提がまだ弱い

- Hosting の設定や deploy 手順が未整備
- 環境変数の扱いがローカル前提で、公開時のセットアップが README からは分かりにくい
- 監視、運用、障害時の切り戻しなどの観点は未整理

### 3. 品質保証はまだ最小限

- 画面単位の自動テストは未整備
- emulator を使った Rules 検証はあるが、UI とデータの結合検証は人手依存
- bundle size warning のような改善余地が残っている

## 公開前に見ておくべきポイント

### 優先度高

- Firebase 環境変数の管理方法を README / deploy 手順へ明記する
- SPA rewrite を含む hosting 設定を確定する
- 認証済み / 未認証の導線を本番 URL で再確認する
- 画像や bundle size の重さを最低限見直す

### 優先度中

- README を「概要」「起動」「検証」「docs 参照」に整理する
- 初回デプロイ後の確認チェックリストを作る
- Firestore / Auth / Storage の本番設定差分を洗い出す

### 優先度低

- UI の磨き込み
- 状態管理やコンポーネント責務の再整理
- 将来向けの通知、検索、タグ機能などの拡張

## 判定

現状の `Real Chat` は「ローカルで動く学習用プロトタイプ」としては十分に成立しています。

一方で、「公開して継続運用するアプリ」としては、まだ次の整理が必要です。

- deploy 手順の確立
- 公開後の最低限の確認手順
- ドキュメントの入口整理
- 軽微な品質改善

## 次フェーズの位置づけ

次にやるべきことは大きく 3 系統です。

1. ドキュメントと振り返りを整えて、現状を言語化する
2. デプロイ手順を整理して、初回公開できる状態にする
3. 公開後に残る改善候補を backlog として切り出す
